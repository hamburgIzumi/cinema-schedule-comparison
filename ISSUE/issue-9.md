# [Issue #9] デモ（ダミー）データ表示時の明確化表示・警告バッジの実装

- **ステータス**: 完了 (Completed)
- **担当**: 役割B (フロントエンド開発) / 役割D (デザイナー)
- **ブランチ**: `feat/issue-9-fallback-badge` -> `work`

## 1. 目的
通信遮断やサイト構造変更時にフォールバック（デモ/ダミーデータ）が表示された際、ユーザーが実際の最新データなのかデモデータなのかを明確に識別できるようにUI上で明示する。

## 2. 実装内容
- `js/fetchers/*.js`: 各フェッチャーで抽出成功時は `isFallback: false`、フォールバック時は `isFallback: true` をデータ構造に付与
- `css/components.css`: 警告バナーおよび `⚠️ デモ（ダミー）データ` バッジのスタイリング
- `js/uiRender.js`: 映画館ヘッダーおよびステータスバーへの「デモデータ表示中」明示タグ・注記の動的追加
- `js/app.js`: データ取得完了時のステータス表示の更新（デモデータが含まれる場合の警告メッセージ表示）

## 3. 完了条件
- [x] デモデータが表示されている映画館ヘッダーに `⚠️ デモ（ダミー）データ` が明示されること
- [x] ステータスバーおよび空席詳細モーダルにデモデータである旨の注意書きが表示されること
- [x] 実データ取得成功時には警告が表示されないこと
