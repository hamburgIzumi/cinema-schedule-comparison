# [Issue #8] 各映画館の直接API/データフェッチ方式への切り替えと切り戻し用安全スイッチの実装

- **ステータス**: 完了 (Completed)
- **担当**: 役割B (フロントエンド開発)
- **ブランチ**: `feat/issue-8-api-fetchers` -> `work`

## 1. 目的
公式サイトとの最新データ一致率を高めるため、従来のHTML直接パースに加え、各映画館サイトの内部API / スケジュールJSONデータフェッチ処理を第一優先で実行するロジックを実装する。
また、万が一通信障害やAPI変更が発生した場合に備え、設定フラグ1つで従来の取得方式や元の状態にすぐ切り戻せる安全設計（トグルスイッチ機能）を組み込む。

## 2. 実装内容
- `config/cinemas.json`: `useDirectApi` (boolean) および `apiEndpoints` などの取得切り替えフラグを追加
- `js/fetchers/tohoFetcher.js`: TOHOシネマズ用API / 直接データ構造解析ロジックの優先実行
- `js/fetchers/aeonFetcher.js`: イオンシネマ用API / 直接データ構造解析ロジックの優先実行
- `js/fetchers/tokyu109Fetcher.js`: 109シネマズ用API / 直接データ構造解析ロジックの優先実行
- 多重フォールバック構造の構築 (①APIフェッチ -> ②従来のHTMLパース -> ③安全フォールバック)

## 3. 完了条件
- [x] 設定ファイルの `useDirectApi` フラグでAPI取得と従来取得を簡単に切り替えられること
- [x] 映画館の最新実データがAPI / 最適化フェッチで正常に取得できること
- [x] 切り戻しフラグ（`useDirectApi: false`）をセットした際に以前の動作に即座に戻せること
